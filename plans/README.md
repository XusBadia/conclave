## Findings considered and rejected

(So nobody re-audits these.)

- **Batch fail-fast "race" between two cancel flags**: false — the per-case
  closure's `cancel` is a clone of the shared `state.batch_cancel` Arc
  (commands.rs:3210→3224); transport-failure fail-fast works as designed.
- **`BatchDone` emitted before per-case futures finish**: false —
  `.buffer_unordered(..).collect().await` (commands.rs:~3345) is a completion
  barrier; `BatchDone` fires strictly after all outcomes.
- **Stale per-case cancel flags poisoning the next run**: benign —
  `case_cancels.insert()` replaces any stale flag with a fresh `false` on
  every run; worst case is a small map entry leak after a panic (noted in
  plan 002's maintenance notes).
- **`preview_data_boundary` missing CLI ids from the bypass list**: not a
  bug — its `unwrap_or_default()` arm tolerates missing keys by design
  (previews never fail on auth); plan 004 still unifies it for the wasted
  keychain hit.
- **`@react-pdf/renderer` bloating the main bundle**: false — both export
  paths load it via dynamic `import()` (`exportCaseVerdict.tsx:33`,
  `exportCasesBatch.tsx:74`); already code-split.
- **OAuth double-start clobbering the pending Anthropic flow**: last-write-
  wins is defensible UX (the user pastes the code from the most recent tab);
  Anthropic's state check safely rejects the stale code. Not worth changing.
- **Image attachments bypassing the data boundary**: verified enforced —
  `boundary_preview_for_request` blocks cloud-vision sends outside
  `explicit_phi`+consent, and `run_draft_case` rebuilds `attached_file_paths`
  from stored rows (commands.rs:2542) so drafts are covered too. A regression
  test would be welcome inside the golden-cases work (unplanned item 2).
- **CLI subprocess env-var allowlist hardening**: marginal — `CLAUDE_*`/
  `ANTHROPIC_*`/`CODEX_*`/`OPENAI_*` are already scrubbed and prompts run in
  a TempDir with tools disabled; an attacker who controls the user's
  environment already owns the account. Revisit only if the threat model
  changes.
- **fastembed model-cache integrity checks**: marginal — cache lives in a
  user-private directory; a corrupted model fails loudly. Not worth a custom
  checksum layer.
- **Frontend major-version bumps (React 19 / Vite 7 / Tailwind 4 / i18next 24)**:
  deliberate deferral — pre-1.0 product, working build, no EOL pressure;
  batch them when there's a feature reason.
- **Duplicate transitive `rand`/`sha2`/`thiserror` versions in Cargo.lock**:
  benign Rust ecosystem reality (Lance/Arrow/DataFusion pull their own);
  binary-size cost only.
- **Embedding/retrieval LRU cache**: premature — LLM latency dominates
  wall-clock; revisit with profiling evidence.
- **`run_deliberation` forward_task hang on error paths**: unverified
  speculation (MED-confidence auditor claim, no reproduction); the channel
  sender drops when `run_deliberation` returns. Investigate only if a stuck
  "Running…" banner is observed without a panic in logs.
